# Contributing

Use `.ai/` as the source of truth for planning, workflow, memory, and evaluation artifacts.

Before submitting implementation changes:

- Read the active workstream under `.ai/PLANS/current-sprint/`.
- Run the relevant guard scripts before mutating shell commands or production implementation files.
- Run `scripts/verify.sh` and the project smoke command when available.
- Record blockers, accepted risks, and validation evidence in the relevant sprint artifact.
