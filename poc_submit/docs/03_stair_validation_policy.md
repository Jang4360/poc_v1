# 계단 후보 검증 정책

계단 데이터는 수치지형도 계단 객체를 그대로 확정 데이터로 보지 않는다. 보행 경로에 영향을 줄 가능성이 높은 후보만 지도 검토 대상으로 둔다.

## 최종 지도 표시 기준

P1/P2만 기본 채택 후보로 표시한다.

| 우선순위 | 개수 | 기준 |
|---|---:|---|
| P1 | 666 | 도로 polygon과 겹치거나 5m 이내라 실제 보행 경로에 걸릴 가능성이 큼 |
| P2 | 124 | 도로 20m 이내이면서 횡단보도 30m 또는 편의시설 50m 단서가 있음 |
| 합계 | 790 | 지도 기본 표시 대상 |

P3/P4는 이번 제출 지도에서는 표시하지 않는다. 기존 산출물은 시행착오와 추적을 위해 보존하되, 팀 공유용 지도에서는 P1/P2만 사용한다.

## 로드뷰 AI 검증 폐기 사유

초기에는 카카오 로드뷰를 8방향으로 캡처하고 AI로 계단 여부를 판정하려 했다. 그러나 로드뷰 위치와 계단 객체 위치가 어긋나거나, 로드뷰가 없는 구간이 많아 결측/오판정이 많았다. 따라서 현재는 로드뷰 전수 검증보다 거리 기반 후보 축소와 수동 검토를 우선한다.
