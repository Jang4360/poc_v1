# 지도 레이어 UX 기준

## 기본값

팀 공유용 기본 화면은 해운대구 기준으로 시작한다.

| 레이어 | 기본값 | 이유 |
|---|---|---|
| 계단 후보 | ON | P1/P2 채택 후보를 먼저 검수해야 함 |
| 횡단보도 | ON | 계단 후보와 보행 연결성을 같이 보기 위함 |
| 편의시설 | ON | 보행약자 목적지와 접근성 정보를 같이 보기 위함 |
| 계단 원천 | OFF | 원천 전체는 후보 검수 시 화면을 복잡하게 만듦 |
| 도로 경사·노면·폭 | OFF | 분석용 대용량 레이어라 필요할 때만 선택 구군 단위로 로드 |
| 도로면 polygon | OFF | 비교/검증용 레이어라 기본 표시하지 않음 |

## 왼쪽 패널

왼쪽은 조작만 담당한다.

| 섹션 | 포함 레이어/필터 |
|---|---|
| 계단 검수 | 계단 후보 토글, P1~P4 필터, 계단 원천 토글 |
| 기본 참조 | 횡단보도, 편의시설, 편의시설 분류 |
| 도로 분석 | 경사·노면·폭, 도로면 polygon, 보행 등급/경사/노면/폭 필터 |

상단 상태칩은 현재 화면에 실제로 표시되는지 알 수 있게 `OFF`를 명시한다.

## 오른쪽 범례

오른쪽은 해석만 담당한다.

| 섹션 | 역할 |
|---|---|
| 레이어 운영 기준 | 기본 ON/OFF와 구군 단위 로드 원칙 설명 |
| 점 / 아이콘 | 축척에 따른 표시 방식 설명 |
| 계단 후보 검증 | P1~P4 판정 기준 설명 |
| 도로 경사/노면/폭 | 색상, 선 모양, 굵기 의미 설명 |
| 편의시설 | 장소 category와 접근성 featureType 분리 기준 설명 |

## 운영 원칙

도로 계열 레이어는 전체 부산을 한 번에 띄우지 않고 선택 구군만 로드한다. 계단 검수는 기본적으로 P1/P2만 보며, P3/P4는 검증 범위를 넓힐 때만 켠다. 편의시설은 색을 카테고리별로 쪼개지 않고 파란색으로 통일해 접근성 배지와 충돌하지 않게 한다.
