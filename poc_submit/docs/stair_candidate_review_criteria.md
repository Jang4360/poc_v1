# 계단 후보 검증 기준

## 목적

수치지형도 계단 객체를 그대로 서비스 경로에 반영하지 않고, 도로 polygon, 횡단보도, 보행약자 편의시설과의 근접성을 기준으로 우선 검증 대상을 정한다.

## 현재 기준 데이터

| 항목 | 값 |
|---|---:|
| 부산 전체 계단 후보 | 3,744 |
| 해운대구 계단 후보 | 297 |
| 해운대구 기본 검수 대상 | 74 |

`index.html`은 `assets/data/busan-stair-review-candidates-data.js`를 기준으로 표시한다.

## 우선순위 기준

| 우선순위 | 서비스 판정 | 조건 | 해운대구 개수 | 반영 방침 |
|---|---|---|---:|---|
| `P1` | 채택 후보 | 도로 polygon 5m 이내 또는 겹침 | 53 | 기본 표시, 우선 검증 |
| `P2` | 채택 후보 | 도로 polygon 20m 이내이고, 횡단보도 30m 이내 또는 편의시설 50m 이내 | 21 | 기본 표시, 우선 검증 |
| `P3` | 보류 | 도로 polygon 20m 이내이나 횡단보도/편의시설 근접 단서 없음 | 38 | 기본 미표시, 필요 시 보조 검증 |
| `P4` | 제외/참고 | 도로 polygon 20m 초과 | 185 | 서비스 경로 반영 제외, 참고 데이터로 보관 |

## 채택 원칙

- `P1`과 `P2`만 기본 검수 대상으로 본다.
- `P1`과 `P2`도 확정 계단이 아니라 채택 후보이므로, 최종 반영 전 현장/로드뷰/팀원 검증이 필요하다.
- `P3`은 도로와는 가깝지만 보행 흐름 단서가 부족하므로 기본 경로 비용에 반영하지 않는다.
- `P4`는 현재 도로 polygon과의 연결 가능성이 낮아 서비스 경로 반영 대상에서 제외한다.
- 중복 클러스터는 하나의 실제 계단 주변에 여러 객체가 있을 수 있으므로, 최종 채택 시 클러스터 단위로 병합 여부를 검토한다.

## 거리 기준

| 단서 | 거리 | 의미 |
|---|---:|---|
| 도로 polygon | 0~5m | 실제 도로/보행 경로에 직접 영향 가능성이 큼 |
| 도로 polygon | 5~20m | 도로 주변 계단일 가능성은 있으나 추가 단서 필요 |
| 횡단보도 | 30m 이내 | 보행 이동 흐름과 연결될 가능성 증가 |
| 편의시설 | 50m 이내 | 보행약자 목적지 주변 계단일 가능성 증가 |
