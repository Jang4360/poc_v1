## 1. 프로젝트 개요

### 1.1 배경 및 목적

부산시는 언덕·계단·복잡한 지형이 많아 휠체어 이용자, 고령자, 유아차 동반 시민 등 이동 약자에게 특히 불편한 도시 구조를 가지고 있다. 기존 네이버·카카오맵 등의 길찾기 서비스는 일반 보행자 기준으로 경로를 제공하며, 무장애(Barrier-Free) 이동 경로에 대한 정보가 부재하거나 부정확하다.

**부산이음길**은 이동 약자가 안전하고 독립적으로 부산 곳곳을 이동할 수 있도록 무장애 경로 정보를 제공하는 길찾기 서비스이다.

### 1.2 서비스명

- **정식 명칭 (가칭):** 부산이음길
- **영문명 (가칭):** Busan EumGil
- **의미:** 부산의 교통약자를 '이어주는 길', 장벽 없이 연결되는 도시

### 1.3 목표

1. 부산 내 무장애 이동 경로 데이터베이스 구축
2. 이동 약자 맞춤 경로 탐색 기능 제공
3. 시민 참여형 장애물 정보 업데이트 체계 마련
4. 부산시 교통·복지 정책과의 연계 기반 마련

## 2. 타겟 사용자

> 기능명세서 기준으로 장애 유형을 **시각장애** / **보행약자** 두 가지로 통일한다.
> 보행약자에는 휠체어 이용자, 고령자, 유아차 동반 보호자, 일시적 이동 불편자 등이 포함된다.
> 본 서비스는 일반 보행자용 범용 길찾기 서비스가 아니라 교통약자 전용 접근성 길찾기 서비스다.

| 사용자 그룹 | 분류 | 설명 |
|---|---|---|
| 시각장애인 | 시각장애 | 음성 안내, 점자블록 정보 중심. LLM 대화형 UI는 추후 확장 검토 |
| 휠체어 이용자 | 보행약자 | 전동/수동 휠체어 사용자, 경사로·엘리베이터 필수 |
| 고령자 | 보행약자 | 계단·경사면 이동 어려움 |
| 유아차 동반 보호자 | 보행약자 | 단차·계단 회피 경로 필요 |
| 일시적 이동 불편자 | 보행약자 | 골절·수술 후 회복 중인 시민 |

## 3. 핵심 기능 요구사항

### 3.0 앱 초기 설정

#### 3.0.1 온보딩 *(필수)*
- 첫 접속 사용자에게 온보딩 화면을 제공한다.
- 장애 유형 선택: **시각장애** / **보행약자**
- 장애 등급 선택
- 온보딩 완료 후 위치정보 이용약관 동의 단계로 이동
- 로그인 없이도 설정 가능 (로컬 저장)

#### 3.0.2 위치정보 이용약관 동의 *(필수)*
- 온보딩 완료 후 앱 최초 진입 시 1회 필수로 위치정보 이용약관 동의를 받는다.
- 로그인 여부와 무관하게 앱 삭제 후 재설치 시 재동의 필요.
- 법적 필수 사항이며 서비스 이용의 전제 조건이다.

---

### 3.1 유저 정보

> 현재 프로젝트 구현은 비로그인(local-first)을 원칙으로 하며, 로그인 없이도 모든 기본 기능을 이용 가능하다.
> 북마크·자주 가는 길 등 저장 데이터는 현재 구현 범위에서는 로컬 스토리지에 저장한다. 로그인 및 계정 연동은 추후 확장으로 검토한다.

#### 3.1.1 회원가입 *(추후 확장)*
- 닉네임, 장애유형, 장애등급, 전화번호를 입력하여 계정 생성
- 닉네임은 개인정보 보호를 위해 실명 대신 사용
- 전화번호는 본인 또는 보호자 번호 입력 (응급상황 대비); 미입력 시 119 기본 설정

#### 3.1.2 로그인 *(추후 확장)*
- 소셜 로그인: 카카오, 네이버, 구글
- 계정 기반 데이터 동기화 및 개인화 기능과 연동

#### 3.1.3 로그아웃 / 회원탈퇴 *(추후 확장)*
- 로그아웃 시 로컬 데이터는 유지
- 회원탈퇴 시 모든 저장 데이터 삭제; 탈퇴 후 재가입 가능하나 기존 데이터 복구 불가

---

### 3.2 지도 (메인)

#### 3.2.1 현재 위치 표시 *(필수)*
- 앱 실행 시 사용자의 현재 위치를 지도에 표시
- 위치 권한이 허용된 경우에만 작동; 실시간 위치 업데이트

#### 3.2.2 베리어프리 시설 정보 표시 *(필수)*
- 지도에 아래 시설 위치를 마커로 표시하며, 각 시설별 아이콘을 다르게 표시한다.
  - **음식점** (베리어프리 인증)
  - **관광지** (베리어프리 인증)
  - **장애인 화장실**
  - **엘리베이터**
  - **전동휠체어 충전소**
- 마커 클릭 시 상세정보 표시 (이름, 주소, 베리어프리 시설 여부 — 경사로, 자동문 유무 등)
- 공공 데이터 및 무장애부산관광 데이터 연계

#### 3.2.3 점자블록 위치 표시 *(필수)*
- 점자블록 위치를 지도에 표시 (데이터 수집 예정)
- 시각장애인 이동 보조 목적

#### 3.2.4 필터 기능 *(필수)*
- 시설 유형 필터 옵션: 음식점, 관광지, 화장실, 엘리베이터, 충전소
- 다중 선택 가능; 필터 적용 시 해당 마커만 표시

#### 3.2.5 지도 확대/축소/이동 *(선택)*
- 핀치 제스처로 확대/축소, 드래그로 이동
- 내 위치 버튼으로 현재 위치 복귀

---

### 3.3 검색

#### 3.3.1 목적지 검색 (텍스트) *(필수)*
- 장소명, 주소 입력으로 목적지 검색
- 검색 결과 리스트 표시 (이름, 주소, 거리); 부산 지역 우선 검색
- 검색 결과 선택 시 지도에 해당 위치 표시

#### 3.3.2 목적지 검색 (음성) *(선택)*
- 시각장애인 모드에서 STT 기반 음성 목적지 검색
- 마이크 버튼 터치로 음성 입력 시작/종료
- 인식된 텍스트로 자동 검색 실행 (표준어 기반; 사투리 인식은 추후 고려)

#### 3.3.3 최근 검색 기록 *(선택)*
- 최근 검색 장소 최대 10개 저장 (로컬); 개별 삭제 가능

---

### 3.4 내비게이션

#### 3.4.1 경로 안내 *(필수)*
- 출발지·도착지 설정 시 보행약자 이용 가능한 최적 경로 안내
- 경사로·엘리베이터·자동문 있는 경로 우선; 계단·급경사 회피
- 예상 소요 시간 표시; 경로를 지도에 시각적으로 표시
- 휠체어 최적 경로 알고리즘 및 도로 경사도 데이터 필요

#### 3.4.2 음성 안내 (TTS) *(필수)*
- 내비게이션 중 음성으로 경로 안내 ("100미터 전방에서 좌회전하세요" 등)
- 음성 안내 ON/OFF 설정 가능

#### 3.4.3 경로 이탈 시 재탐색 *(선택)*
- 안내 경로 이탈 시 자동 경로 재탐색
- 재탐색 중 음성/자막 안내 제공

---

### 3.5 경로 수집

#### 3.5.1 사용자 경로 데이터 수집 *(필수)*
- 사용자 동의 하에 실제 이용한 경로 수집
- 수집 데이터: GPS 좌표, 이동 시간, 장애 유형
- 설정에서 데이터 수집 ON/OFF 가능 (익명화 처리)
- 수집된 데이터는 서비스 개선 및 경로 최적화에 활용

#### 3.5.2 자주 가는 길 저장 *(필수)*
- 자주 이용하는 경로 저장 및 빠른 검색
- 현재 구현 범위에서는 로컬 저장

### 3.6 제보 / 신고

#### 3.6.1 도로 상태 제보 *(필수)*

### 3.9 시각장애인 전용 UI

#### 3.9.1 대화형 LLM 인터페이스 *(선택)*
- 시각장애인 모드에서 LLM 기반 대화형 UI 제공
- 음성 상호작용 버튼 (큰 버튼, 노란색 계열)
- 사용자 음성 입력: "OOO로 가고 싶어" → LLM이 의도 파악
- LLM 음성 응답 확인 후 자동으로 현재 위치 기반 경로 안내 시작
- STT + LLM + TTS 통합; 대화 히스토리 관리

#### 3.9.2 리스트 기반 화면 구성 *(선택)*
- 지도 대신 장소 목록을 리스트 형식으로 제공
- 최소 폰트 18pt, 버튼 최소 60×60pt, 노란색 계열 배경색
- 화면당 최대 3개 버튼으로 구성

#### 3.9.3 TalkBack 지원 *(선택)*
- 모든 UI 요소에 명확한 접근성 라벨 지정
- 화면 읽기 순서 최적화
- Android TalkBack 기준 접근성 확보

#### 3.9.4 음성 브리핑 강화 *(선택)*
- 내비게이션 시작 전 전체 경로 음성 브리핑 제공
- "총 거리 500미터, 예상 시간 10분, 좌회전 2회, 우회전 1회" 등
- 브리핑 재생/일시정지 기능

## 6. 사용자 시나리오 (User Story)

### 시나리오 1: 전동휠체어 이용자의 병원 방문
> "나는 전동휠체어를 이용하는 60대 남성이다. 부산대학교병원에서 부산역까지 혼자 이동하고 싶다. 계단이 없고 경사가 완만한 길로 안내받고 싶다."

**흐름:** 앱 실행 → 온보딩(보행약자 선택) → 위치정보 이용약관 동의 → 출발지/목적지 입력 → 경로 탐색 → 엘리베이터 경유 경로 안내 → 도착

### 시나리오 2: 비로그인 사용자의 목적지 탐색
> "나는 처음 가는 장소를 찾아야 하는 이동 약자다. 로그인 없이도 주변 시설 정보를 확인하고 목적지까지 길안내를 받고 싶다."

**흐름:** 앱 실행 → 온보딩 → 위치정보 이용약관 동의 → 지도/검색에서 목적지 탐색 → 시설 상세 확인 → 경로 설정 → 음성 안내 시작

### 시나리오 3: 시민의 장애물 신고
> "오늘 보도에 공사 펜스가 설치되어 길이 막혔다. 다른 이동 약자들이 위험할 것 같아 신고하고 싶다."
