# 📋 북마크 API 명세

> **작성일:** 2026-04-12  
> **작성자:** 김응서  
> **최종 수정일:** 2026-04-12

---

## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `GET /bookmarks` | 저장한 장소 목록 조회 |
| P1 | `POST /bookmarks` | 장소 북마크 등록. 외부 검색 장소는 `places` 저장 후 북마크 |
| P1 | `DELETE /bookmarks/places/{placeId}` | 장소 북마크 해제 |

> 북마크 서버 API는 로그인 사용자의 서버 저장 데이터만 처리한다. 비로그인 사용자는 이 API를 호출하지 않고 앱 로컬 저장소에서 처리한다.

## 2. 저장한 장소 목록 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/bookmarks` |
| 인증 | 필요 |
| 설명 | 로그인 사용자가 서버에 저장한 장소 목록을 조회한다. 비로그인 사용자의 저장 장소는 클라이언트 로컬 저장소에서 관리한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| page | int | N | 페이지 번호 |
| size | int | N | 페이지 크기 |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| content | array | N | 북마크 장소 목록 |
| content[].bookmarkId | Long | N | 북마크 ID |
| content[].placeId | Long | N | 장소 ID |
| content[].providerPlaceId | string | Y | 카카오 장소 ID |
| content[].name | string | N | 장소명 |
| content[].category | string | N | 장소 카테고리 |
| content[].address | string | Y | 주소 |
| content[].point | object | N | 장소 좌표 |
| page | int | N | 현재 페이지 번호 |
| size | int | N | 페이지 크기 |
| totalElements | Long | N | 전체 데이터 수 |
| totalPages | int | N | 전체 페이지 수 |
| hasNext | boolean | N | 다음 페이지 존재 여부 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "content": [
      {
        "bookmarkId": 1,
        "placeId": 10,
        "providerPlaceId": "123456789",
        "name": "부산시민공원",
        "category": "TOURIST_SPOT",
        "address": "부산광역시 부산진구 시민공원로 73",
        "point": {
          "lat": 35.1686,
          "lng": 129.0576
        }
      }
    ],
    "page": 0,
    "size": 10,
    "totalElements": 1,
    "totalPages": 1,
    "hasNext": false
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 401 | UNAUTHORIZED | 인증이 필요합니다. | 토큰 누락/만료 |

## 3. 장소 북마크 등록

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/bookmarks` |
| 인증 | 필요 |
| 설명 | 로그인 사용자의 장소를 서버 북마크로 등록한다. 내부 장소는 `placeId`로 등록하고, 외부 검색 장소는 `providerPlaceId` 기준으로 `places`를 조회하거나 생성한 뒤 북마크한다. 비로그인 사용자의 북마크는 클라이언트 로컬 저장소에 저장한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| placeId | Long | N | 내부 장소 ID. 값이 있으면 이 장소를 바로 북마크 |
| providerPlaceId | string | N | 카카오 장소 ID. `placeId`가 없을 때 필수 |
| name | string | N | 장소명. `placeId`가 없을 때 필수 |
| category | string | N | 장소 카테고리. `placeId`가 없을 때 필수 |
| address | string | N | 주소 |
| point | object | N | 장소 좌표. `placeId`가 없을 때 필수 |
| point.lat | double | N | 위도 |
| point.lng | double | N | 경도 |

#### Request Example

내부 장소 북마크:

```json
{
  "placeId": 10
}
```

외부 검색 장소 북마크:

```json
{
  "providerPlaceId": "987654321",
  "name": "서면역",
  "category": "BUS_STATION",
  "address": "부산광역시 부산진구 중앙대로 지하 730",
  "point": {
    "lat": 35.1577,
    "lng": 129.0590
  }
}
```

### Response

- HTTP Status: `201 Created`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| bookmarkId | Long | N | 북마크 ID |
| placeId | Long | N | 장소 ID |
| providerPlaceId | string | Y | 카카오 장소 ID |
| createdPlace | boolean | N | 외부 장소 요청으로 `places`가 새로 생성되었는지 여부 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "bookmarkId": 1,
    "placeId": 10,
    "providerPlaceId": "123456789",
    "createdPlace": false
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `placeId`와 `providerPlaceId`가 모두 없거나, 외부 장소 저장에 필요한 `name`, `category`, `point` 누락 |
| 401 | UNAUTHORIZED | 인증이 필요합니다. | 토큰 누락/만료 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | `placeId`로 요청한 내부 장소가 존재하지 않음 |
| 409 | CONFLICT | 이미 처리된 요청입니다. | 이미 북마크한 장소 |

### 처리 규칙

- `placeId`가 있으면 내부 `places.placeId`를 기준으로 북마크한다.
- `placeId`가 없고 `providerPlaceId`가 있으면 `places.providerPlaceId`로 기존 장소를 조회한다.
- `providerPlaceId`로 조회되지 않으면 요청의 `name`, `category`, `address`, `point`로 `places`를 먼저 생성하고, 생성된 `placeId`로 북마크한다.
- 서버 저장 시 `bookmarks`에는 최종적으로 UUID 기반 `userId`, `placeId`만 저장한다.
- 비로그인 사용자는 서버 북마크를 생성하지 않고 클라이언트 로컬 저장소에 저장한다.

## 4. 장소 북마크 해제

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `DELETE` |
| URL | `/bookmarks/places/{placeId}` |
| 인증 | 필요 |
| 설명 | 로그인 사용자의 서버 북마크를 장소 ID 기준으로 해제한다. 비로그인 사용자의 북마크 해제는 클라이언트 로컬 저장소에서 처리한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| placeId | Long | Y | 북마크 해제할 장소 ID |

#### Body

> 없음

### Response

- HTTP Status: `204 No Content`
- `data = null`

#### Success Example

```json
{
  "success": true,
  "data": null,
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `placeId` 형식 오류 |
| 401 | UNAUTHORIZED | 인증이 필요합니다. | 토큰 누락/만료 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | 북마크 내역이 존재하지 않음 |
