## 2. 소셜 로그인

### 기본 정보

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/auth/social-login` |
| 인증 | 불필요 |
| 설명 | 소셜 제공자 토큰으로 로그인한다. 최초 로그인 사용자는 프로필 미완성 상태로 처리하고 `profileCompleted=false`를 반환한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| socialProvider | string | Y | 소셜 제공자. `KAKAO`, `NAVER`, `GOOGLE` |
| socialAccessToken | string | Y | 소셜 제공자에서 발급받은 액세스 토큰 |

#### Request Example

```json
{
  "socialProvider": "KAKAO",
  "socialAccessToken": "kakao-access-token"
}
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| accessToken | string | N | 서비스 액세스 토큰 |
| refreshToken | string | N | 서비스 리프레시 토큰 |
| userId | string(UUID) | N | 사용자 ID |
| isNewUser | boolean | N | 최초 가입 여부 |
| profileCompleted | boolean | N | 필수 프로필 입력 완료 여부 |

#### 비고

- 신규 사용자는 이 응답만으로 회원가입 완료 상태가 아니다.
- 신규 사용자는 `users` row를 생성하되, 필수 프로필 입력 전까지 `profileCompleted=false` 상태로 둔다.
- `PATCH /users/me`에서 `nickname`, `disabilityType`을 저장해야 회원가입 완료 상태가 된다.
- `profileCompleted=false`인 사용자는 프로필 입력 화면으로 이동시키며, 지도/경로 탐색 등 서비스 핵심 기능에 진입할 수 없다.
- API 응답과 JWT subject에 사용하는 사용자 식별자는 UUID 문자열이다.
- 클라이언트 요청에서 임의의 `userId`를 받아 사용자 권한을 판단하지 않는다. 내 정보 처리는 토큰 기준의 `/users/me`로 처리한다.

#### Success Example

```json
{
  "success": true,
  "data": {
    "accessToken": "access-token",
    "refreshToken": "refresh-token",
    "userId": "018f7f6c-2b7e-7c3a-9f4a-8b4e3b7c9a01",
    "isNewUser": true,
    "profileCompleted": false
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `socialProvider`, `socialAccessToken` 누락 또는 형식 오류 |
| 401 | UNAUTHORIZED | 인증에 실패했습니다. | 소셜 액세스 토큰이 유효하지 않음 |
| 502 | EXTERNAL_API_ERROR | 외부 API 호출에 실패했습니다. | 소셜 제공자 API 호출 실패 |

## 3. 토큰 재발급

### 기본 정보

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/auth/reissue` |
| 인증 | 불필요 |
| 설명 | 리프레시 토큰으로 액세스 토큰을 재발급한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| refreshToken | string | Y | 리프레시 토큰 |

#### Request Example

```json
{
  "refreshToken": "refresh-token"
}
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| accessToken | string | N | 새 액세스 토큰 |
| refreshToken | string | N | 새 리프레시 토큰 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "accessToken": "new-access-token",
    "refreshToken": "new-refresh-token"
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `refreshToken` 누락 |
| 401 | UNAUTHORIZED | 인증이 필요합니다. | 리프레시 토큰 만료 또는 위조 |
