## 1) Request

### 1.1 Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

> 인증 불필요 API는 Authorization 항목 제거

### 1.2 Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| {id} | Long | Y | 리소스 식별자 |

> 없으면 `없음`

### 1.3 Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| page | int | N | 페이지 번호 |
| size | int | N | 페이지 크기 |

> 없으면 `없음`

### 1.4 Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| fieldA | boolean | Y | 필드 설명 |
| fieldB | string | N | 필드 설명 |

> 없으면 `없음`

### 1.5 Request Example

```json
{
  "fieldA": true,
  "fieldB": "value"
}
```

---

## 2) Response

### 2.1 Success

- HTTP Status: `200 OK` / `201 Created` / `204 No Content`

### 2.2 Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| id | Long | N | 리소스 식별자 |
| name | string | N | 리소스 이름 |

> `204` 또는 데이터 없음인 경우: `data = null`

### 2.3 Success Example

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "example"
  },
  "message": null
}
```

---

## 3) Error

### 3.1 Error Code

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. (또는 검증 메시지) | 요청값 검증 실패 |
| 401 | UNAUTHORIZED | 인증이 필요합니다. | 토큰 누락/만료 |
| 403 | FORBIDDEN | 권한이 없습니다. | 권한 없음 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | 조회 대상 없음 |
| 500 | INTERNAL_ERROR | 서버 내부 오류가 발생했습니다. | 서버 예외 |
| 409 | CONFLICT | 이미 처리된 요청입니다. | 중복/상태 충돌 |
| 502 | EXTERNAL_API_ERROR | 외부 API 호출에 실패했습니다. | 카카오 API 등 실패 |

### 3.2 Error Example

```json
{
  "success": false,
  "code": "EXTERNAL_API_ERROR",
  "message": "외부 API 호출에 실패했습니다."
}
```
