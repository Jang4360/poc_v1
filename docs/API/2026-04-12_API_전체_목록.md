## 3. API 목록

| 우선순위 | 대분류 | 소분류 | API | 포함 기능 | 판단 |
| --- | --- | --- | --- | --- | --- |
| P1 | 사용자 도메인 | 인증 | `POST /auth/social-login` | 소셜 로그인, 최초 가입 처리 | 소셜 토큰으로 로그인한다. 신규 사용자는 프로필 미완성 상태로 처리하고 `profileCompleted=false` 반환 |
| P1 | 사용자 도메인 | 인증 | `POST /auth/reissue` | 토큰 재발급 | 유지 |
| P1 | 사용자 도메인 | 인증 | `POST /auth/logout` | 로그아웃 | 유지 |
| P2 | 사용자 도메인 | 인증 | `DELETE /users/me` | 회원탈퇴 | 기능명세서에는 있음. MVP 화면에서는 제외지만 개인정보 기능이라 후보로 둠 |
| P1 | 사용자 도메인 | 사용자 정보/설정 | `GET /users/me` | 내 정보 조회 | 유지 |
| P1 | 사용자 도메인 | 사용자 정보/설정 | `PATCH /users/me` | 닉네임, 장애유형, 장애등급, 전화번호 수정 | 신규 사용자는 닉네임과 장애유형을 입력해야 회원가입 완료. 닉네임은 중복 허용 |
| P1 | 사용자 도메인 | 사용자 정보/설정 | `PATCH /users/me/settings` | 위치 약관 동의, TTS, 경로 데이터 수집 동의, 푸시 알림 설정 | 설정 관련 기능을 하나로 합침 |
| P1 | 사용자 도메인 | 북마크 | `GET /bookmarks` | 저장한 장소 목록 조회 | 로그인 사용자는 서버 DB 조회. 비로그인 사용자는 API 호출 없이 앱 로컬 저장소 조회 |
| P1 | 사용자 도메인 | 북마크 | `POST /bookmarks` | 장소 북마크 등록 | 로그인 사용자는 서버 DB 저장. 비로그인 사용자는 API 호출 없이 앱 로컬 저장소 저장 |
| P1 | 사용자 도메인 | 북마크 | `DELETE /bookmarks/places/{placeId}` | 장소 북마크 해제 | 로그인 사용자는 서버 DB 삭제. 비로그인 사용자는 API 호출 없이 앱 로컬 저장소 삭제 |
| P1 | 사용자 도메인 | 자주 가는 길 | `GET /favorite-routes` | 자주 가는 길 목록 조회 | 로그인 사용자는 서버 DB 조회. 비로그인 사용자는 API 호출 없이 앱 로컬 저장소 조회 |
| P1 | 사용자 도메인 | 자주 가는 길 | `POST /favorite-routes` | 자주 가는 길 저장 | 로그인 사용자는 서버 DB 저장. 비로그인 사용자는 API 호출 없이 앱 로컬 저장소 저장 |
| P1 | 사용자 도메인 | 자주 가는 길 | `PATCH /favorite-routes/{favRouteId}` | 자주 가는 길 수정 | 로그인 사용자는 서버 DB 수정. 비로그인 사용자는 API 호출 없이 앱 로컬 저장소 수정 |
| P1 | 사용자 도메인 | 자주 가는 길 | `DELETE /favorite-routes/{favRouteId}` | 자주 가는 길 삭제 | 로그인 사용자는 서버 DB 삭제. 비로그인 사용자는 API 호출 없이 앱 로컬 저장소 삭제 |
| P1 | 사용자 도메인 | 제보 | `POST /hazard-reports` | 도로 상태 제보 등록 | 익명 제보로 등록. 사용자 계정과 연결하지 않음 |
| P1 | 장소 도메인 | 장소/지도 | `GET /places` | 주변 시설 마커 조회, 카테고리 필터 | `lat`, `lng`, `radius`, `category` query로 합침 |
| P1 | 장소 도메인 | 장소/지도 | `GET /places/{placeId}` | 시설 상세 조회, 접근성 정보 조회 | `place_accessibility_features`까지 응답에 포함 |
| P1 | 장소 도메인 | 장소 검색 | `GET /places/search` | 텍스트 장소 검색 | 카카오 API 연동, 내부 `places.providerPlaceId` 매칭 |
| P2 | 장소 도메인 | 장소 검색 | `POST /places/search/voice` | 음성 검색 STT | 음성 파일 업로드는 제외하고 `recognizedText` 기반으로 검색 |
| P1 | 보행 네트워크 도메인 | 경로 | `POST /routes/search` | 경로 탐색, 안전/최단/대중교통 옵션 비교, 경로 구간 정보, 위험도 정보, TTS 안내 문구 | Query Parameter 없음. Body의 `routeOption`은 단일 선택값이며, 없으면 `SAFE`, `SHORTEST`, `PUBLIC_TRANSPORT` 전체 반환 |
| P1 | 보행 네트워크 도메인 | 경로 로그 | `POST /route-logs` | 실제 이동 GPS 로그 수집 | 인증 불필요. 내비게이션 종료 시점에 사용자/기기 식별자 없이 GPS 로그 일괄 업로드. 추천 활용은 후순위 |
| P2 | 보행 네트워크 도메인 | 경로 | `POST /routes/reroute` | 경로 이탈 재탐색 | 현재 위치, 도착지, 기존 옵션 기준으로 재탐색 |
| P2 | 대중교통 도메인 | 버스 정류소 | `GET /public-transit/bus/stops` | 버스 정류소 검색 | 부산 BIMS 정류소 식별자 확보용. 정류소명/ARS 번호 중심, 주변 정류장 후보는 카카오 장소 검색 또는 정류소 캐시와 매칭 |
| P2 | 대중교통 도메인 | 버스 노선 | `GET /public-transit/bus/routes/{routeId}/stops` | 노선 정류장 순서 및 실시간 버스 위치 조회 | 부산 BIMS `busInfoByRouteId` 연동. 직행/환승 구간 검증에 사용 |
| P2 | 대중교통 도메인 | 버스 도착 정보 | `GET /public-transit/bus/stops/{busStopId}/arrivals` | 정류소 도착 정보 조회 | 부산 BIMS `stopArrByBstopid` 연동. `routeId` query가 있으면 특정 노선만 필터링 |
| P2 | 대중교통 도메인 | 접근성 경로 | `POST /public-transit/accessible-route-candidates` | 접근성 대중교통 경로 후보 조회 | ODsay 등 외부 대중교통 길찾기 API와 부산 BIMS/부산교통공사 데이터를 조합. Version A 직행, Version B 환승 포함 PoC 필요 |
| P1 | 운영 도메인 | Slack 제보 검토 | `POST /slack/hazard-reports/actions` | Slack 제보 승인/반려 처리 | Slack 버튼 콜백으로 `hazard_reports.status`를 `APPROVED` 또는 `REJECTED`로 변경 |

## 5. 공통 응답 형식

### Success

```json
{
  "success": true,
  "data": {},
  "message": null
}
```

### Error

```json
{
  "success": false,
  "code": "INVALID_INPUT",
  "message": "잘못된 입력입니다."
}
```

## 6. 공통 Error Code

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | 요청값 검증 실패 |
| 401 | UNAUTHORIZED | 인증이 필요합니다. | 토큰 누락/만료 |
| 403 | FORBIDDEN | 권한이 없습니다. | 권한 없음 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | 조회 대상 없음 |
| 409 | CONFLICT | 이미 처리된 요청입니다. | 중복/상태 충돌 |
| 500 | INTERNAL_ERROR | 서버 내부 오류가 발생했습니다. | 서버 예외 |
| 502 | EXTERNAL_API_ERROR | 외부 API 호출에 실패했습니다. | 카카오 API 등 외부 API 실패 |
