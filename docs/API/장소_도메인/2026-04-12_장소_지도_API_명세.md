## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `GET /places` | 주변 시설 마커 조회, 카테고리 필터 |
| P1 | `GET /places/{placeId}` | 시설 상세 조회, 접근성 정보 조회 |

---

## 2. 주변 시설 마커 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/places` |
| 인증 | 선택 |
| 설명 | 현재 지도 영역 또는 현재 위치 기준으로 주변 시설 마커 목록을 조회한다. 카테고리 필터를 query parameter로 함께 처리한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | N | Bearer 액세스 토큰. 로그인 사용자의 북마크 여부가 필요한 경우 사용 |
| Content-Type | Y | `application/json` |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| lat | double | Y | 기준 위도 |
| lng | double | Y | 기준 경도 |
| radius | int | N | 검색 반경. meter 단위 |
| category | string | N | 장소 카테고리. 다중 선택 시 comma-separated 값 사용. `RESTAURANT`, `TOURIST_SPOT`, `TOILET`, `BUS_STATION`, `ELEVATOR`, `CHARGING_STATION`, `BARRIER_FREE_FACILITY`, `ACCOMMODATION` |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| places | array | N | 장소 목록 |
| places[].placeId | Long | N | 장소 ID |
| places[].name | string | N | 장소명 |
| places[].category | string | N | 카테고리 |
| places[].address | string | Y | 주소 |
| places[].point | object | N | 좌표 |
| places[].point.lat | number | N | 위도 |
| places[].point.lng | number | N | 경도 |
| places[].isBookmarked | boolean | N | 현재 사용자 북마크 여부. 비로그인 시 `false` |

#### Success Example

```json
{
  "success": true,
  "data": {
    "places": [
      {
        "placeId": 10,
        "name": "부산시민공원",
        "category": "TOURIST_SPOT",
        "address": "부산광역시 부산진구 시민공원로 73",
        "point": {
          "lat": 35.1686,
          "lng": 129.0576
        },
        "isBookmarked": false
      }
    ]
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `lat`, `lng`, `radius`, `category` 형식 오류 |

## 3. 시설 상세 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/places/{placeId}` |
| 인증 | 선택 |
| 설명 | 시설 상세 정보와 장소 접근성 속성을 함께 조회한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | N | Bearer 액세스 토큰. 로그인 사용자의 북마크 여부가 필요한 경우 사용 |
| Content-Type | Y | `application/json` |

#### Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| placeId | Long | Y | 장소 ID |

#### Query Parameter

> 없음

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| placeId | Long | N | 장소 ID |
| name | string | N | 장소명 |
| category | string | N | 카테고리 |
| address | string | Y | 주소 |
| point | object | N | 좌표 |
| providerPlaceId | string | Y | 카카오 장소 ID |
| accessibilityFeatures | array | N | 장소 접근성 속성 목록. 접근성 정보가 없으면 빈 배열 `[]` |
| accessibilityFeatures[].featureType | string | N | 접근성 속성 유형 |
| accessibilityFeatures[].available | boolean | N | 제공 여부 |
| isBookmarked | boolean | N | 현재 사용자 북마크 여부. 비로그인 시 `false` |

#### Success Example

```json
{
  "success": true,
  "data": {
    "placeId": 10,
    "name": "부산시민공원",
    "category": "TOURIST_SPOT",
    "address": "부산광역시 부산진구 시민공원로 73",
    "point": {
      "lat": 35.1686,
      "lng": 129.0576
    },
    "providerPlaceId": "123456789",
    "accessibilityFeatures": [
      {
        "featureType": "ramp",
        "available": true
      },
      {
        "featureType": "accessible_toilet",
        "available": true
      }
    ],
    "isBookmarked": false
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `placeId` 형식 오류 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | 장소가 존재하지 않음 |

### 비고

- 운영시간은 DB에 저장하지 않는다.
- 카카오 API에서 운영시간을 제공하거나 별도 외부 데이터가 있는 경우에만 응답 확장 후보로 둔다.
