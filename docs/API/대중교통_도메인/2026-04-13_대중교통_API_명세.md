> **작성자:** 김응서

## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P2 | `GET /public-transit/bus/stops` | 버스 정류소 검색 |
| P2 | `GET /public-transit/bus/routes/{routeId}/stops` | 노선 정류장 순서 및 실시간 버스 위치 조회 |
| P2 | `GET /public-transit/bus/stops/{busStopId}/arrivals` | 정류소 도착 정보 조회 |
| P2 | `POST /public-transit/accessible-route-candidates` | 접근성 대중교통 경로 후보 조회 |

## 2. 외부 연동 기준

| 구분 | 내용 |
| --- | --- |
| 외부 API | ODsay 대중교통 길찾기 API, 부산광역시_부산버스정보시스템, 부산교통공사 공공데이터 |
| 제공기관 | ODsay, 부산광역시, 부산교통공사 |
| 주요 용도 | 대중교통 경로 후보 생성, 버스 저상 여부 검증, 지하철 시간표/역 접근성 보강 |
| 데이터 포맷 | 외부 API별 상이. 내부 응답은 JSON으로 통일 |

### 외부 API 매핑

| 우리 API | 외부 연동 | 비고 |
| --- | --- | --- |
| `GET /public-transit/bus/stops` | 부산 BIMS `busStopList` | ARS 번호 또는 정류장명으로 정류소 조회. 주변 정류장 후보는 카카오 장소 검색 또는 정류소 캐시와 BIMS 정류소를 매칭 |
| `GET /public-transit/bus/routes/{routeId}/stops` | 부산 BIMS `busInfoByRouteId` | 노선 정류소 조회, 실시간 버스 위치, `lowplate` 포함 |
| `GET /public-transit/bus/stops/{busStopId}/arrivals` | 부산 BIMS `stopArrByBstopid` 또는 `busStopArrByBstopidLineid` | 정류소 ID 기준 도착 정보. `routeId` query가 있으면 특정 노선 도착 정보로 필터링. `lowplate1`, `lowplate2` 포함 |
| `POST /public-transit/accessible-route-candidates` | ODsay 대중교통 길찾기 + 부산 BIMS + 부산교통공사 공공데이터 | 대중교통 경로 후보는 ODsay에서 받고, 접근성 검증은 우리 서버에서 수행 |

### PoC 검증 기준

| 버전 | 범위 | 검증할 내용 | 판단 |
| --- | --- | --- | --- |
| Version A | 직행 검증형 | ODsay가 직행 버스/직행 지하철 후보를 반환하는지, 버스 후보의 정류소/노선 식별자로 부산 BIMS 저상버스 검증이 가능한지, 지하철 후보에 시간표/역 접근성 정보를 보강할 수 있는지 확인 | 가능하면 P2 구현 |
| Version B | 환승 포함 검증형 | ODsay 환승 후보의 모든 보행 구간을 우리 보행 경로로 재검증할 수 있는지, 여러 버스 구간의 저상 여부와 여러 지하철역 접근성을 모두 검증할 수 있는지 확인 | 검증 불완전하면 참고 후보 또는 HOLD |
| 공통 | 식별자 매칭 | ODsay의 버스 정류소/노선 식별자가 부산 BIMS의 `bstopid`, `lineid`와 안정적으로 매칭되는지 확인 | 매칭이 불안정하면 BIMS 정류소 도착 정보 전체 조회 후 노선 번호/도착 시간 기준으로 보수적으로 매칭 |

### 저상버스 필드 기준

- 부산 BIMS의 `lowplate`, `lowplate1`, `lowplate2` 값은 `0: 일반`, `1: 저상`으로 해석한다.
- 우리 API 응답에서는 `isLowFloor` boolean으로 변환한다.
- 앞차/뒷차가 나뉘는 도착 정보는 `firstArrival`, `secondArrival`로 변환한다.
- 일반 버스도 도착 정보 화면에는 표시할 수 있지만, 교통약자 경로 추천 후보에는 `isLowFloor=true`인 차량만 사용한다.
- 환승 경로는 ODsay에서 후보를 받을 수 있으나, 모든 보행/버스/지하철 구간의 접근성 검증이 가능하다는 PoC 결과가 나오기 전까지 자동 추천 확정 후보로 사용하지 않는다.

## 3. 버스 정류소 검색

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P2 |
| Method | `GET` |
| URL | `/public-transit/bus/stops` |
| 인증 | 불필요 |
| 설명 | 부산 BIMS 도착 정보 조회에 필요한 정류소 식별자를 확보한다. 정류소명 또는 ARS 번호로 검색하며, 주변 정류장 후보는 카카오 장소 검색 또는 정류소 캐시와 BIMS 정류소를 매칭한다. |

### Request

#### Query Parameter

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| name | string | N | 정류소명 |
| arsNo | string | N | 정류소 ARS 번호 |
| lat | number | N | 주변 정류장 후보 조회 기준 위도 |
| lng | number | N | 주변 정류장 후보 조회 기준 경도 |
| radius | int | N | 주변 정류장 후보 조회 반경. 단위 meter |
| page | int | N | 페이지 번호. 기본값 `1` |
| size | int | N | 페이지 크기. 기본값 `10` |

#### Request Example

```http
GET /public-transit/bus/stops?name=부산시청&page=1&size=10
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| stops | array | N | 정류소 목록 |
| stops[].busStopId | string | N | 정류소 ID |
| stops[].arsNo | string | N | 정류소 ARS 번호 |
| stops[].name | string | N | 정류소명 |
| stops[].lat | number | N | 위도 |
| stops[].lng | number | N | 경도 |
| stops[].stopType | string | N | 정류장 구분 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "stops": [
      {
        "busStopId": "505780000",
        "arsNo": "13708",
        "name": "부산시청",
        "lat": 35.179937855685,
        "lng": 129.07691415917,
        "stopType": "일반"
      }
    ]
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `name`, `arsNo`, `lat/lng/radius` 조건이 모두 없음 |
| 502 | EXTERNAL_API_ERROR | 외부 API 호출에 실패했습니다. | 부산 BIMS 또는 장소 검색 API 호출 실패 |

## 4. 노선 정류장 순서 및 실시간 버스 위치 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P2 |
| Method | `GET` |
| URL | `/public-transit/bus/routes/{routeId}/stops` |
| 인증 | 불필요 |
| 설명 | 특정 노선의 정류장 순서와 실시간 버스 위치를 조회한다. 탑승 정류장과 하차 정류장이 같은 노선에 있고 순서상 직행 가능한지 검증하는 데 사용한다. |

### Request

#### Path Variable

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| routeId | string | Y | 노선 ID |

#### Request Example

```http
GET /public-transit/bus/routes/5200179000/stops
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| routeId | string | N | 노선 ID |
| stops | array | N | 노선 정류장 목록 |
| stops[].sequence | int | N | 정류장 순서 |
| stops[].busStopId | string | N | 정류소 ID |
| stops[].arsNo | string | Y | 정류소 ARS 번호 |
| stops[].name | string | N | 정류소명 |
| stops[].lat | number | Y | 위도 |
| stops[].lng | number | Y | 경도 |
| stops[].isTurnPoint | boolean | Y | 회차지 여부 |
| vehicles | array | N | 실시간 차량 위치 목록 |
| vehicles[].vehicleId | string | Y | 차량 식별자 |
| vehicles[].currentStopSequence | int | Y | 차량이 위치한 정류장 순서 |
| vehicles[].isLowFloor | boolean | Y | 저상버스 여부 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "routeId": "5200179000",
    "stops": [
      {
        "sequence": 1,
        "busStopId": "505780000",
        "arsNo": "13708",
        "name": "부산시청",
        "lat": 35.179937855685,
        "lng": 129.07691415917,
        "isTurnPoint": false
      }
    ],
    "vehicles": [
      {
        "vehicleId": "부산70자0000",
        "currentStopSequence": 3,
        "isLowFloor": true
      }
    ]
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | `routeId` 값 오류 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | 해당 노선을 찾을 수 없음 |
| 502 | EXTERNAL_API_ERROR | 외부 API 호출에 실패했습니다. | 부산 BIMS 호출 실패 |

```http
GET /public-transit/bus/stops/505780000/arrivals
```

## 6. 접근성 대중교통 경로 후보 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P2 |
| Method | `POST` |
| URL | `/public-transit/accessible-route-candidates` |
| 인증 | 불필요 |
| 설명 | 출발지와 도착지를 기준으로 보행-only 경로보다 빠른 접근성 대중교통 경로 후보를 조회한다. 대중교통 후보는 ODsay에서 받고, 보행/저상버스/지하철 접근성 검증은 우리 서버에서 수행한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| startPoint | object | Y | 출발지 좌표 |
| startPoint.lat | number | Y | 출발지 위도 |
| startPoint.lng | number | Y | 출발지 경도 |
| endPoint | object | Y | 도착지 좌표 |
| endPoint.lat | number | Y | 도착지 위도 |
| endPoint.lng | number | Y | 도착지 경도 |
| disabilityType | string | Y | 장애 유형. `VISUAL`, `MOBILITY` |
| maxWalkMeterToStation | int | N | 출발지/도착지에서 정류소 또는 역까지 허용할 최대 보행 거리. 기본값 `500` |
| includeTransferCandidates | boolean | N | 환승 후보 포함 여부. 기본값 `false`. PoC 전까지는 참고 후보로만 사용 |
| maxCandidates | int | N | 최대 후보 개수. 기본값 `3` |

#### Request Example

```json
{
  "startPoint": {
    "lat": 35.1796,
    "lng": 129.0756
  },
  "endPoint": {
    "lat": 35.1577,
    "lng": 129.0590
  },
  "disabilityType": "MOBILITY",
  "maxWalkMeterToStation": 500,
  "includeTransferCandidates": true,
  "maxCandidates": 3
}
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| candidates | array | N | 접근성 대중교통 경로 후보 목록 |
| candidates[].type | string | N | 후보 유형. `LOW_FLOOR_BUS_DIRECT`, `SUBWAY_DIRECT`, `TRANSFER_PUBLIC_TRANSIT` |
| candidates[].verificationStatus | string | N | 접근성 검증 상태. `VERIFIED`, `PARTIAL`, `UNVERIFIED` |
| candidates[].arrivalPredictionType | string | N | 도착 예측 기준. `REALTIME`, `TIMETABLE`, `EXTERNAL_ESTIMATE` |
| candidates[].estimatedTotalMinute | int | N | 전체 예상 소요 시간 |
| candidates[].walkingOnlyEstimatedMinute | int | N | 보행-only 예상 소요 시간 |
| candidates[].savedMinute | int | N | 보행-only 대비 절약 시간 |
| candidates[].totalWalkDistanceMeter | number | N | 전체 보행 거리 |
| candidates[].transferCount | int | N | 환승 횟수 |
| candidates[].segments | array | N | 구간 목록 |
| candidates[].segments[].type | string | N | 구간 유형. `WALK`, `BUS`, `SUBWAY` |
| candidates[].segments[].accessibilityVerified | boolean | N | 구간 접근성 검증 여부 |
| candidates[].segments[].distanceMeter | number | Y | 구간 거리 |
| candidates[].segments[].durationMinute | int | Y | 구간 예상 소요 시간 |
| candidates[].segments[].routeId | string | Y | 버스/지하철 노선 ID |
| candidates[].segments[].routeNo | string | Y | 버스 번호 |
| candidates[].segments[].lineName | string | Y | 지하철 노선명 |
| candidates[].segments[].isLowFloor | boolean | Y | 버스 구간 저상버스 여부 |
| candidates[].segments[].waitMinute | int | Y | 버스/지하철 대기 시간 |
| candidates[].segments[].startStationName | string | Y | 탑승 정류소/역 이름 |
| candidates[].segments[].endStationName | string | Y | 하차 정류소/역 이름 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "candidates": [
      {
        "type": "LOW_FLOOR_BUS_DIRECT",
        "verificationStatus": "VERIFIED",
        "arrivalPredictionType": "REALTIME",
        "estimatedTotalMinute": 24,
        "walkingOnlyEstimatedMinute": 34,
        "savedMinute": 10,
        "totalWalkDistanceMeter": 390,
        "transferCount": 0,
        "segments": [
          {
            "type": "WALK",
            "accessibilityVerified": true,
            "distanceMeter": 210,
            "durationMinute": 5
          },
          {
            "type": "BUS",
            "accessibilityVerified": true,
            "durationMinute": 15,
            "routeId": "5200179000",
            "routeNo": "128-1",
```

### 비고

- 대중교통 경로 후보 생성은 직접 구현하지 않고 ODsay 같은 외부 대중교통 길찾기 API를 우선 사용한다.
- 보행-only 예상 시간은 서버가 기존 보행 경로 탐색 로직으로 계산한다. 클라이언트가 넘긴 값만 신뢰하지 않는다.
- Version A는 직행 검증형이다. `LOW_FLOOR_BUS_DIRECT`, `SUBWAY_DIRECT` 후보에서 출발지-탑승지, 하차지-도착지 보행 구간을 기존 보행 경로 탐색 기준으로 검증한다.
- Version B는 환승 포함 검증형이다. `TRANSFER_PUBLIC_TRANSIT` 후보에서 처음/중간/끝 보행 구간을 모두 기존 보행 경로 탐색 기준으로 재검증한다.
- 버스 구간은 부산 BIMS로 저상버스 여부를 검증한다. 일반 버스는 도착 정보 화면에는 표시할 수 있으나, 접근성 추천 확정 후보에서는 제외한다.
- 지하철 구간은 시간표 기반으로 도착 시간을 예측하고, 역 접근성 정보를 보강한다. 실시간 도착처럼 표현하지 않는다.
- 보행-only 예상 시간보다 느린 대중교통 경로는 후보에서 제외한다.
- 환승 후보는 모든 보행/버스/지하철 구간의 접근성 검증이 끝나기 전까지 `PARTIAL`로 분리 표시한다.
- 저상버스 예약은 백엔드 API로 제공하지 않는다. 필요하면 프론트에서 부산시버스정보시스템 교통약자 화면을 외부 링크로 직접 연결한다.
